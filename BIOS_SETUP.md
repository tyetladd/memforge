# Подготовка BIOS / UEFI для загрузки MemForge

## Secure Boot — теперь работает БЕЗ отключения

С версии v0.4 флешка использует **PreLoader.efi** от Linux Foundation
(подписан Microsoft Corporation UEFI CA 2011 — той же подписью, которой
доверяет каждый UEFI BIOS из коробки). Schema загрузки:

```
BIOS → BOOTX64.EFI (PreLoader, MS-signed) → loader.efi (MemForge2)
                          ↓
            (один раз на каждом ПК) HashTool → enroll → reboot
```

### Первая загрузка на каждом новом ПК (одноразово, ~30 секунд)

1. Воткни флешку, выбери её в Boot Menu (см. таблицу клавиш ниже)
2. Загружается **PreLoader** → не находит хэш MemForge в MOK List → запускает HashTool
3. В HashTool:
   - **"Enroll Hash"**
   - Выбрать **`loader.efi`** в списке файлов
   - **"Continue boot"** → перезагрузка
4. После reboot снова выбери флешку в Boot Menu → MemForge запустится без вопросов

### Все последующие загрузки на этом же ПК

Просто Boot Menu → USB → MemForge стартует напрямую. Хэш сохранён в NVRAM,
PreLoader проверяет один раз и пропускает.

> ✓ Secure Boot **остаётся включённым**.
> ✓ BitLocker НЕ запросит recovery key (мы не трогаем PCR конфигурацию).
> ✓ Win 11 не требует отключения CSM/Secure Boot.
> ✓ Не нужен BIOS-пароль / админ — всё делается из меню загрузки.

### Если PreLoader не запустился

- Старые BIOS (до 2012) могут не иметь Microsoft UEFI CA — там придётся
  всё-таки отключить Secure Boot (см. ниже).
- Некоторые корпоративные политики удаляют Microsoft CA из db — тоже
  потребуется отключить SB.

В таких случаях см. секцию **«Отключение Secure Boot»** ниже.

# Старый путь (отключить Secure Boot) — если PreLoader не работает

## 1. Войти в BIOS / UEFI Setup

При включении ПК — нажать одну из клавиш:

| Вендор | Клавиша |
|---|---|
| Dell, HP, Lenovo (ноуты) | **F2** или **F12** |
| ASUS, ASRock, Gigabyte, MSI | **Del** (иногда F2) |
| Apple Mac | удерживать **Option (⌥)** |
| Surface | удерживать **Volume Up** при включении |

---

## 2. Отключить Secure Boot

Найти раздел **Security** или **Boot** → **Secure Boot** → **Disabled**.

После выключения иногда требуется ещё раз сохранить и зайти, чтобы появились
дополнительные опции типа «Delete all secure boot variables».

> ⚠ Если на ноутбуке/ПК включён **BitLocker** — выключение Secure Boot приведёт
> к запросу **Recovery Key** при следующей загрузке Windows. Выпишите ключ
> заранее: **Параметры → Конфиденциальность и защита → Шифрование устройства →
> Управление ключами восстановления BitLocker**.

---

## 3. Включить загрузку с USB

| Раздел | Что включить |
|---|---|
| **Boot Mode** | **UEFI** (не Legacy и не CSM-only) |
| **CSM (Compatibility Support Module)** | можно оставить включённым, но не обязателен |
| **Fast Boot / Fast Startup** | **Disabled** — иначе USB не успевает подцепиться |
| **USB Boot** | **Enabled** |

---

## 4. Однократная загрузка с флешки

После сохранения BIOS-настроек (F10):

* Перезагрузить.
* На POST-экране нажать **Boot Menu** клавишу:

| Вендор | Клавиша |
|---|---|
| Dell | **F12** |
| HP | **F9** |
| Lenovo | **F12** или Fn+F12 |
| ASUS, MSI, Gigabyte | **F11** или **F8** |
| Apple Mac | удерживать **⌥ (Option)** |

* Выбрать USB-устройство с пометкой **UEFI:** (например, `UEFI: SanDisk Cruzer`).

---

## 5. Что не поддерживается

| Платформа | Причина |
|---|---|
| **Apple Silicon (M1/M2/M3)** | x86_64-only бинарь, не запустится |
| **ARM-ноуты** (Snapdragon X, Surface Pro X) | то же — другая архитектура |
| **Старые офисные ПК с Legacy BIOS** (до ~2012) | нет UEFI вообще; используйте Memtest86+ через GRUB |
| **Серверы без графической консоли** (только IPMI/iLO/serial) | UEFI-меню рендерится через GOP, на serial-консоль не выводится |

---

## 6. Особенности по типам железа

### Ноутбуки

* **Подключите зарядку.** Тест на CPU 90% разрядит батарею за 30-60 минут.
* Тонкие ультрабуки (XPS 13, MacBook Air, Yoga Slim) уйдут в тепловой троттлинг через ~30 секунд AVX2.
* Если на ноутбуке гибридная графика (Intel iGPU + Nvidia/AMD dGPU) — UEFI-меню может уйти на «не тот» дисплей. В BIOS попробовать переключить **MUX / Graphics Switch → Integrated only**.

### Игровые ПК

* Если включён **XMP** или **EXPO**, тест может показать ошибки из-за высоких частот, а не из-за плохой памяти. **Перед тем как грешить на железо** — сбросьте профиль на JEDEC defaults, прогоните тест ещё раз.
* На Intel 12-14 поколения **Big.LITTLE**: проверьте, что в логе строка `Cores: ...` показывает все ядра (P + E). Если не все — обновите BIOS материнки.
* На больших RAM (64-128 ГБ) один проход занимает 10-15 минут — это нормально.

### Серверы / рабочие станции

* **NUMA**: если 2+ сокета, проверьте что тест задействует все ноды (логи `Cores: %d total, %d enabled`).
* **ECC RAM** молча исправляет single-bit ошибки — тест может показать PASS при реально слабых ячейках. Это ограничение тестера, не лечится без правки бинаря.

## 7. После теста — где смотреть результаты

После каждого прогона прога пишет два файла **в корень флешки** (рядом с `EFI/`):

* **`memforge2.log`** — полный текстовый лог: пройденные тесты, найденные ошибки,
  SPD каждой планки с серийником и таймингами, MCA-диффы, снимки температуры/
  напряжения на момент каждой ошибки.
* **`report.json`** — те же данные в структурированном виде, плюс блок `peaks`
  с пиковой температурой / мощностью / частотой за прогон. Удобно для
  скармливания внешнему анализатору.

Файлы перезаписываются при каждом следующем прогоне — забирайте сразу или
делайте копию перед повторным запуском.

> ⚠ **Сам UEFI-тест памяти на тестируемой машине интернета не требует** — он
> работает полностью офлайн. Анализ логов (если используете) делается на
> отдельном ПК.
